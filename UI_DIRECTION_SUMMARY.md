# UI Direction Summary

## Recommended direction
**Soft Utility**

A calm, Apple-native, object-centered design language for a footwear intelligence app.

## Should feel like
- trustworthy
- human
- calm
- useful
- native to iPhone
- grounded in real objects and real wear

## Should not feel like
- a running app
- a performance dashboard
- a sneaker resale app
- an AI gimmick product

## Core references
- Apple Health for trust and clarity
- Gentler Streak for humane tone
- AllTrails for calm detail confidence
- gear/inventory apps for object history thinking

## Core implementation principles
- grouped list/detail structure
- restrained cards
- low chart dependence
- plain-language summaries
- semantic status over fake precision

## Design application status
This direction has now been applied across the live prototype surfaces for:
- onboarding
- Home
- Footwear list
- Footwear detail
- Assign
- condition check-in
- Insights
- Settings
- create-footwear flow

## Key product-level rule
The app tracks footwear wear and condition in everyday life.
It should not drift into elite fitness culture or quantified-self theater.
